📘 HAVING Clause - Detailed Notes

1. What is HAVING?

The HAVING clause is used to filter grouped data after applying aggregate functions.

👉 Works after GROUP BY

2. HAVING Clause Syntax

SELECT column, AGG_FUNCTION(column)
FROM table
GROUP BY column
HAVING condition;

3. Aggregation with HAVING

Used to filter groups based on aggregate values.

SELECT dept_id, AVG(salary)
FROM employees
GROUP BY dept_id
HAVING AVG(salary) > 5000;

👉 Returns departments where average salary is greater than 5000

4. HAVING without GROUP BY

HAVING can be used without GROUP BY when aggregation is applied to the entire table.

SELECT AVG(salary)
FROM employees
HAVING AVG(salary) > 5000;

👉 Treats entire table as a single group

5. HAVING vs WHERE

  • WHERE: Filters rows before grouping
  • HAVING: Filters after grouping

6. Restriction on Grouped Queries

  • Columns in SELECT must be:
    • Either in GROUP BY
    • Or used with aggregate functions
  • Cannot select non-grouped, non-aggregated columns
-- ❌ Invalid
SELECT dept_id, name, AVG(salary)
FROM employees
GROUP BY dept_id; -- ✅ Valid
SELECT dept_id, AVG(salary)
FROM employees
GROUP BY dept_id;

7. Combined Example

SELECT dept_id, COUNT(*)
FROM employees
WHERE salary > 3000
GROUP BY dept_id
HAVING COUNT(*) > 5;

👉 Filters rows → groups → filters groups

8. Summary

  • HAVING filters grouped data
  • Used with aggregate functions
  • Works after GROUP BY
  • Can be used without GROUP BY
