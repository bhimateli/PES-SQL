📘 SQL Joins - Complete Guide

1. Introduction to Joins

Joins are used to combine data from multiple tables based on a related column.

2. ER Diagram (Concept)

Shows relationship between tables (e.g., Employee → Department using foreign key).

3. Simple Join

SELECT e.name, d.dept_name FROM emp e JOIN dept d ON e.dept_id = d.id;

4. Table Aliases

Short names for tables.

SELECT e.name FROM emp e;

5. Multi-table Join

SELECT e.name, d.dept_name, l.city FROM emp e JOIN dept d ON e.dept_id = d.id JOIN location l ON d.loc_id = l.id;

6. Joins with GROUP BY

SELECT d.dept_name, COUNT(e.id) FROM emp e JOIN dept d ON e.dept_id = d.id GROUP BY d.dept_name;

7. JOIN with WHERE / ON / USING

-- ON SELECT * FROM emp e JOIN dept d ON e.dept_id = d.id; -- WHERE SELECT * FROM emp e, dept d WHERE e.dept_id = d.id; -- USING SELECT * FROM emp JOIN dept USING (dept_id);

8. Join with Row Selection

SELECT e.name FROM emp e JOIN dept d ON e.dept_id = d.id WHERE d.dept_name = 'HR';

9. Natural Join

Automatically joins columns with same name.

SELECT * FROM emp NATURAL JOIN dept;

10. Equi Join

Join using equality condition (=).

SELECT * FROM emp e JOIN dept d ON e.dept_id = d.id;

11. Non-Equi Join

Join using non-equality conditions.

SELECT * FROM emp e JOIN salary_grade s ON e.salary BETWEEN s.min_sal AND s.max_sal;

12. Left, Right, Outer Join

-- LEFT JOIN SELECT * FROM emp e LEFT JOIN dept d ON e.dept_id = d.id; -- RIGHT JOIN SELECT * FROM emp e RIGHT JOIN dept d ON e.dept_id = d.id; -- FULL OUTER JOIN SELECT * FROM emp e FULL OUTER JOIN dept d ON e.dept_id = d.id;

13. Cross Join

Returns all combinations.

SELECT * FROM emp CROSS JOIN dept;

14. Rules for Multi-Join Queries

  • Use proper join conditions to avoid Cartesian product
  • Use table aliases for readability
  • Join order matters for performance
  • Filter data using WHERE clause

15. Summary

  • Joins combine data from multiple tables
  • Different types serve different purposes
  • Use aliases for clarity
  • Always define proper join conditions
