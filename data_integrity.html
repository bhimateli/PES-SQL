📘 Database Concepts Cheat Sheet

1. Normalization

Normalization is the process of organizing data to reduce redundancy.

  • 1NF: Remove duplicate columns, atomic values
  • 2NF: Remove partial dependency
  • 3NF: Remove transitive dependency

2. Column Constraints

Rules applied on columns to ensure valid data.

  • NOT NULL – value cannot be empty
  • DEFAULT – default value if none provided

3. Check Constraints

Ensures values meet a condition.

CHECK (salary > 0)

4. Uniqueness Constraints

Ensures all values in a column are unique.

UNIQUE (email)

5. NULL Values

Represents missing or unknown data.

  • NULL ≠ 0
  • NULL ≠ empty string

6. Referential Integrity

Ensures relationships between tables remain consistent.

FOREIGN KEY (dept_id) REFERENCES department(id)

7. Referential Integrity Problems

  • Orphan records
  • Invalid foreign keys

8. Delete and Update Rules

  • CASCADE – delete/update child rows
  • SET NULL – set foreign key to NULL
  • RESTRICT – prevent deletion

9. Cascaded Deletes and Updates

Automatically propagate changes to child tables.

ON DELETE CASCADE
ON UPDATE CASCADE

10. Referential Cycles

Occurs when tables reference each other.

Example: A → B → A

11. Foreign Keys and NULL Values

Foreign keys can be NULL unless restricted.

  • NULL means no relationship
  • Useful for optional relationships

12. ACID Properties

  • Atomicity: All or nothing
  • Consistency: Valid state
  • Isolation: Transactions don’t interfere
  • Durability: Data is permanent
