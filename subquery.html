📘 Session 6 - Subqueries

1. Introduction to Subquery

A subquery is a query inside another SQL query.

SELECT name FROM emp WHERE salary > (SELECT AVG(salary) FROM emp);

2. Subqueries with IN, ANY, ALL

SELECT name FROM emp WHERE dept_id IN (SELECT id FROM dept); SELECT name FROM emp WHERE salary > ALL (SELECT salary FROM emp WHERE dept_id = 1); SELECT name FROM emp WHERE salary > ANY (SELECT salary FROM emp WHERE dept_id = 1);

3. Subqueries in WHERE Clause

SELECT name FROM emp WHERE dept_id = (SELECT id FROM dept WHERE dept_name='HR');

4. EXISTS / NOT EXISTS

SELECT name FROM emp e WHERE EXISTS ( SELECT 1 FROM dept d WHERE e.dept_id = d.id );

5. Subquery in FROM Clause

SELECT avg_salary FROM ( SELECT AVG(salary) AS avg_salary FROM emp ) AS temp;

6. Nested Subqueries

SELECT name FROM emp WHERE salary > ( SELECT AVG(salary) FROM emp WHERE dept_id IN ( SELECT id FROM dept WHERE location='NY' ) );

7. Set Comparison

SELECT name FROM emp WHERE salary > ALL (SELECT salary FROM emp WHERE dept_id=2);

8. WITH Clause (CTE)

WITH avg_sal AS ( SELECT AVG(salary) AS avg_salary FROM emp ) SELECT * FROM emp, avg_sal WHERE emp.salary > avg_sal.avg_salary;

9. HAVING with Subquery

SELECT dept_id, AVG(salary) FROM emp GROUP BY dept_id HAVING AVG(salary) > ( SELECT AVG(salary) FROM emp );

10. Scalar Subquery

Returns single value

SELECT name, (SELECT AVG(salary) FROM emp) AS avg_salary FROM emp;

11. Subqueries vs Joins

  • Subqueries → easier to read
  • Joins → faster in many cases

12. Row Valued Expression

SELECT * FROM emp WHERE (dept_id, salary) = (1, 5000);

13. Query Expression

Combination of multiple queries using set operators.

SELECT id FROM A UNION SELECT id FROM B;

14. Important Notes

  • Subqueries can be nested
  • Can return single or multiple values
  • Used in SELECT, WHERE, FROM
  • EXISTS is efficient for large data

15. Summary

  • Subqueries simplify complex queries
  • Useful for filtering and comparison
  • CTE (WITH) improves readability
