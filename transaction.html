📘 Transaction Processing - Detailed Notes

1. What is Transaction Processing?

A transaction is a sequence of operations performed as a single logical unit of work.

  • Either all operations succeed → COMMIT
  • Or all operations fail → ROLLBACK
BEGIN TRANSACTION; UPDATE accounts SET balance = balance - 100 WHERE id = 1; UPDATE accounts SET balance = balance + 100 WHERE id = 2; COMMIT;

2. Transactional Model (ACID)

  • Atomicity: All or nothing
  • Consistency: Data remains valid
  • Isolation: Transactions don’t interfere
  • Durability: Changes are permanent

3. Isolation Levels

Isolation controls how transactions see each other’s data.

  • Read Uncommitted: Dirty reads allowed
  • Read Committed: Only committed data visible
  • Repeatable Read: Same data seen during transaction
  • Serializable: Highest isolation, behaves like sequential execution

Problems Solved

  • Dirty Read
  • Non-repeatable Read
  • Phantom Read
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;

4. Classification of Isolation Model

  • Low Isolation: High performance, less consistency
  • High Isolation: Strong consistency, lower performance

5. SAVEPOINT and RELEASE SAVEPOINT

Used to create checkpoints within a transaction.

BEGIN; SAVEPOINT sp1; UPDATE emp SET salary = 5000 WHERE id = 1; ROLLBACK TO sp1; RELEASE SAVEPOINT sp1; COMMIT;
  • SAVEPOINT: Marks a point
  • ROLLBACK TO: Undo to that point
  • RELEASE: Removes savepoint

6. Locking Levels

Locks prevent conflicts between transactions.

  • Row-level lock: Locks specific rows
  • Page-level lock: Locks a block of rows
  • Table-level lock: Locks entire table

7. Shared and Exclusive Locks

  • Shared Lock (S):
    • Used for reading
    • Multiple transactions allowed
  • Exclusive Lock (X):
    • Used for writing
    • No other transaction allowed
SELECT * FROM emp WITH (NOLOCK); -- Shared/No lock scenario UPDATE emp SET salary = 6000 WHERE id = 1; -- Exclusive lock

8. Summary

  • Transactions ensure data integrity
  • Isolation prevents data conflicts
  • Savepoints allow partial rollback
  • Locks control concurrent access
