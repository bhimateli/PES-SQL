📘 SQL SELECT, Operators & Functions

1. SELECT with Predicates

Used to filter records using conditions.

SELECT * FROM employees WHERE salary > 5000;

2. Comparison Operators

  • =, >, <, >=, <=, !=
SELECT * FROM emp WHERE salary >= 5000;

3. BETWEEN

Filters values within a range.

SELECT * FROM emp WHERE salary BETWEEN 3000 AND 6000;

4. IN

Matches values in a list.

SELECT * FROM emp WHERE dept_id IN (1,2,3);

5. LIKE & Wildcards

  • % → any number of characters
  • _ → single character
SELECT * FROM emp WHERE name LIKE 'A%';

6. LIMIT

Restricts number of rows.

SELECT * FROM emp LIMIT 5;

7. SET Operators

  • UNION
  • UNION ALL
  • INTERSECT
  • MINUS

8. UNION vs UNION ALL

  • UNION → removes duplicates
  • UNION ALL → keeps duplicates
SELECT id FROM A UNION SELECT id FROM B;

9. Duplicate Rows

Handled using DISTINCT.

SELECT DISTINCT dept_id FROM emp;

10. Operator Precedence

Order of execution: NOT → AND → OR

SELECT * FROM emp WHERE salary > 5000 AND dept_id = 1;

11. Numeric Functions

SELECT ROUND(12.567,2); SELECT FLOOR(12.9); SELECT CEIL(12.1); SELECT TRUNCATE(12.567,2);

12. String Functions

SELECT UPPER(name), LOWER(name) FROM emp; SELECT LENGTH(name) FROM emp;

13. Date Functions

SELECT CURRENT_DATE; SELECT YEAR(date), MONTH(date) FROM orders;

14. Other Functions

SELECT BIN(10); SELECT CAST(salary AS CHAR); SELECT COALESCE(NULL, 10);

15. Sorting (ORDER BY)

SELECT * FROM emp ORDER BY salary DESC;
  • ASC → Ascending
  • DESC → Descending

16. Summary

  • Predicates filter data
  • Set operators combine results
  • Functions manipulate data
  • ORDER BY sorts output
