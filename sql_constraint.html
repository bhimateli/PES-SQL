📘 SQL Constraints

1. What are Constraints?

Constraints are rules applied to table columns to ensure data accuracy and integrity.

2. NOT NULL

Ensures that a column cannot have NULL values.

CREATE TABLE emp ( id INT NOT NULL, name VARCHAR(50) );

3. UNIQUE

Ensures all values in a column are different.

CREATE TABLE emp ( email VARCHAR(100) UNIQUE );

4. PRIMARY KEY

Uniquely identifies each record. Cannot be NULL and must be unique.

CREATE TABLE emp ( id INT PRIMARY KEY, name VARCHAR(50) );

5. FOREIGN KEY

Ensures referential integrity between two tables.

CREATE TABLE emp ( dept_id INT, FOREIGN KEY (dept_id) REFERENCES dept(id) );

6. CHECK

Ensures values satisfy a condition.

CREATE TABLE emp ( salary INT CHECK (salary > 0) );

7. DEFAULT

Provides a default value if none is specified.

CREATE TABLE emp ( salary INT DEFAULT 5000 );

8. Combined Example

CREATE TABLE employee ( id INT PRIMARY KEY, name VARCHAR(50) NOT NULL, email VARCHAR(100) UNIQUE, salary INT CHECK (salary > 0), dept_id INT, created_at DATE DEFAULT CURRENT_DATE, FOREIGN KEY (dept_id) REFERENCES department(id) );

9. Summary

  • NOT NULL → No empty values
  • UNIQUE → No duplicates
  • PRIMARY KEY → Unique + Not NULL
  • FOREIGN KEY → Maintains relationship
  • CHECK → Validates condition
  • DEFAULT → Assigns default value
